## PerformanceReport backend

## What to build

Implement the PerformanceReport endpoint and the `PerformanceNarrativeGenerator`.

The report has two sections:

**AI learning section**:
- Last 5 TimedTest results across all Topics (drill-down per Topic shows up to 10 historical results)
- AI-generated narrative produced by `PerformanceNarrativeGenerator`

The narrative is generated by first aggregating four signals as a pure function (testable without LLM):
1. TimedTest scores per Topic
2. ExplanationSession pass counts per Topic (proxy for difficulty)
3. TeacherFlags raised per Topic
4. Activity frequency (session count over a trailing period)

The aggregated signals are passed to the capable-tier LLM to produce a narrative identifying strong Topics, weak Topics, a revision nudge for recently poor-performing Topics, and a comment on study consistency.

**Class Tests section**:
- ScheduledTest results as raw scores only
- No AI narrative; no commentary

## Acceptance criteria

- [ ] `GET /performance/report/{student_id}` returns both sections in a single response
- [ ] AI learning section includes last 5 TimedTest results at the top level; Topic drill-down returns up to 10 results per Topic
- [ ] Signal aggregation is a pure function with no LLM dependency; unit tests cover each signal type from fixture data
- [ ] AI narrative covers: strong Topics, weak Topics, revision nudge, study consistency comment
- [ ] Class Tests section contains ScheduledTest scores only; no AI narrative field is present
- [ ] ScheduledTest results are excluded from signal aggregation and AI narrative generation
- [ ] Narrative generation is invoked at render time (caching is optional for v1)
