# Vidyasaarthi — Agent Context

## What this repo is
This is the documentation and issue tracker for Vidyasaarthi — an AI-powered learning platform for CBSE Classes 8–10 distributed through coaching centers. This repo contains no runnable code. It is the source of truth for specs, architecture decisions, and the project glossary.

## Before doing anything else, read these files
1. `CONTEXT.md` — canonical domain glossary. Every domain term used in issues and ADRs is defined here. Read it fully before reading any issue.
2. `ARCHITECTURE.md` — system component map and key technology decisions.
3. `docs/adr/` — architecture decision records. Read the ADR relevant to whatever you are about to build.
4. The issue spec for the task you have been assigned.

## How to read an issue
Issues follow this structure:
- **What to build** — the feature description and constraints
- **Acceptance criteria** — a checklist; the feature is done when every item is checked

Acceptance criteria are not negotiable. Do not mark a task complete unless every criterion is met.
